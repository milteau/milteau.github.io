随笔：我的 Time-Block Planner (1)

最近的主题阅读，是读 Cal Newport 系列作品，他是一位严谨的计算机科学教授，但是他广为人知的另一面，则是关于生产力领域的深度思考者。与市面上那些充斥着成功学鸡汤的效率书不同，纽波特的理论完全是建立在现代知识工作者的心智模型之上的。在他的逻辑里，心智带宽是一种极其珍贵且易耗的资源。 其实在尝试了无数种诸如番茄工作法、GTD（Getting Things Done）或是各类眼花缭乱的精美手账待办之后，我最钟爱的，依然是 Time-Block （时间块管理）这个看似最传统、实则最硬核的方法。其底层的核心逻辑，用纽波特自己的话来说，就是： Give every minute a job. （给每一分钟派一项工作） 在我的数字日历上，会看到很多块状的格子，上面写得或是看书、或是听播客、或是看YouTube 上面的视频等等。这些不是为了向谁展示的自律标签，而是我个人在信息洪流中为自己亲手拉起的黄色警戒线。 纽波特在《深度工作》（ Deep Work ）中揭示了一个残酷的真相： 知识工作者最大的隐形敌人，不是时间的流逝，而是上下文切换成本（Context-Switching Costs） 。当我们每5分钟忍不住看一眼微信、刷一下微博、抖音、或回一条无关紧要的通知时，我们的大脑并不是在“高效地多线处理”，而是在不同任务的残影之间反复拉扯。每次切换，都会在心智中留下一段“注意力残留（Attention Residue）”，导致我们哪怕坐在桌前，也无法进入真正的神凝状态，即“ 心流 ”。 Time-Block 的精髓，就在于用 空间化 的格子去冻结 线性 的时间。当你给一个两小时的浅蓝色格子冠以“阅读”的头衔时，这意味着你与自己签署了一份具有排他性的绝对契约：在未来的120分钟里，世界与我无关。你不用去纠结“接下来该做什么”，因为时间块已经替你做出了唯一的裁判。忘掉手机，将所有的感官收拢进这一个单一的目标里，心流的诞生才成为可能。

随笔：文本、纸笔与思维的延伸——关于"第一性原则"下的笔记哲学

我们今天的知识生活，似乎正陷入一场前所未有的“工具繁荣”与“思维内耗”。 每隔一段时间，市场上就会涌现出号称能“变革工作流”的笔记软件：有的主打块级数据库，有的主打双向链接图谱，有的则将 AI 无缝塞进每一个弹窗。然而，在体验过无数炫酷的 UI、精心设计的 Card 和 Page 之后，我常常陷入一种微妙的空虚感——我花在研究属性列、调整看板样式、折腾软件插件上的时间，似乎远远超过了我真正用于思考和写作的时间。 当我们剥离这些花哨的包装，重新审视“记录”这件事时，不禁要问：笔记工具的第一性原则，究竟是什么？ 答案也许极其原始：它应当是纯粹文本与载体的统一，是思维无缝延伸的媒介，以及一份能够对抗时间侵蚀的资产。 而沿着这个原则追本溯源，你会发现，无论是在数字世界的极客终点 Emacs，还是在物理世界的古老媒介 实体纸质笔记本（Physical Notebook） 上，它们竟践行着完全同构的哲学。 触感与纯粹：从“被工具驯化”到“回归数据本身” 绝大多数现代笔记软件的逻辑是“应用优先”。它们像是一个个装修精致的房间，你必须按照设计者的逻辑去摆放家具：在 Notion 里，你必须适应 Block；在其他软件里，你必须遵守特定的 Database 架构。你不是在记录思维，你是在填空。 而实体笔记本与 Emacs，在第一性原则上达成了惊人的共识： 载体极简，内容至上 。 一本物理笔记本，只有纸与墨。它没有弹窗，没有格式限制，没有复杂的数据库属性；而在 Emacs 的 Org-mode 里，看似强大的任务跟踪、时间管理和代码块，底层也不过是几行极其朴素的纯文本（Plain Text）。 无论是一张随手翻开的纸页，还是一个 .org 文本文件，它们都将数据与特定应用彻底解耦。三十年过后，服务器或许关停，软件或许过时，但纸页上的笔迹与硬盘里的纯文本，依然清晰可读、字字存真。这种脱离了软件限制的纯粹性，给思维带来了真正的自由。 空间与塑形：不是适应工具，而是打造器官 绝大多数数字工具的寿命，取决于开发者的路线图（Roadmap）。如果官方不支持某个功能，你只能等待，或者忍痛迁移。 但在物理笔记本和 Emacs 的哲学里，工具不是静态的“产品”，而是一个“可编程/可塑造的思维环境”。 纸笔是空间维度上的绝对自由。你可以画大纲，可以涂鸦，可以画思维导图，也可以用“子弹笔记”（Bullet Journal）建立一套属于自己的符号索引。你不需要任何软件更新，就能在下一页纸上创造一种全新的记录逻辑。 而 Emacs 则是在数字维度上继承了这种“纸张般的可塑性”。借助底层的 Elisp，你可以像用泥土捏塑工具一样去塑造它：你需要卡片盒笔记法，它可以演化出 org-roam 或 denote ；你需要时间管理， Org-agenda 早已立于顶峰。 你不是在租赁别人的某种标准工作流，而是在用笔尖或代码，将载体锤炼成最契合你自己大脑突触连接方式的“外挂器官”。工具在适应你，而不是你在适应工具。 零阻力：让落笔追上灵感 思考是极其脆弱的。从“产生想法”到“落笔捕捉”，中间任何微小的阻力——等待软件加载、查找菜单按钮、切换不同应用——都可能让闪现的灵感烟消云散。 实体笔记本的优势在于物理上的“即开即写”：翻开封面，拿起笔，灵感便直达纸面，这种触觉反馈带来的“具身认知”（Embodied Cognition），能让人瞬间进入深度专注。 而 Emacs 带来的则是数字世界里的极致低阻力。在全键盘的操控下，光标可以跟随思维飞驰；其绝妙的全局捕捉机制（ org-capture ），让你无论在处理什么任务，敲击快捷键就能弹出一个极简窗口，记下灵感，瞬间归档，再切回原本的工作。 一个掌控物理空间的触感，一个掌控数字空间的流动，它们都做到了同一种体验：零破裂感。思考未断，笔墨已落。 结语：控制权的回归 有人说，在这个数字化、AI 化的时代，坚持用实体笔记本显得“过于古板”，而折腾 Emacs 门槛又“过于陡峭”。 ...

Markdown 语言学习笔记

这是我学习掌握的第一门轻量级标记语言。 Markdown 是语法简洁，能让人们在进行文字工作时，更关注内容本身而非排版的语言。 值得一提的是，语法极其简单。 要知道——大多数人日常写作只需要 20% 的排版功能（标题、加粗、列表、引用）。Word 提供了 100% 的功能，却让 80% 隐藏在复杂的菜单里；Markdown 只提供最核心的 20%，但让这 20% 的使用效率达到了 100%。 另外一点：人类记忆的最小阻力原则—— 几个符号的记忆成本几乎为零。这种极低的“学习前置成本”，能让初学者在 5 分钟内获得“我掌控了这门工具”的正向反馈。 于我而言，我觉得学计算有关的语言跟学习外语差不多，重点在一个字—— 用 。 代码即语言（Language）： 无论是英语、Python 还是 Markdown，它们的核心功能都是“信息的编码与解码”。 另外，“用”的本质是“情境化学习”： 很多人学不好一门技术，是因为他们试图通过“背诵语法字典”来学习。而真正的掌握，来自于“在输出中输入”。当你为了写一篇今天的日记、记录一个工作流而去敲下 列表 的时候，这个语法就真正长在了你的神经元连接里。

猎乐如猎书：在流媒体时代重返“本地优先”

在这个万物皆可“云端”的时代，我们的精神生活正变得前所未有的便利，却也前所未有的轻飘。打开任何一个主流音乐App，算法像精明的侍者，精准地将符合你过往口味的旋律源源不断地推送到耳边。然而，这种被动接受的愉悦，总让人隐约感到一丝不安。 正是在这种背景下，我选择了一条看似在“开倒车”的路径：将资源端锚定在 Bandcamp ，将播放端留给 foobar2000 。 这不仅是一种技术选择，更是一场关于数字生活的自我重构。 猎乐如猎书：在 Bandcamp 里淘金 Hunting music just like hunting books. 我常常觉得，在流媒体上听歌，就像是在快餐店里吃自助餐，高效却面目模糊；而从本地去淘歌，才真正配得上“捕猎”（Hunting）这个词。 Bandcamp 就是那个独立音乐人的大本营，或者说，它更像是一家开在深夜街角、带着霉味与墨香的独立旧书店。在这里，你不会被大厂的排行榜绑架，你得自己去翻、去听、去偶遇。很多名不见经传的音乐人在这里自由地发布他们充满粗粝感和生命力的作品，并且允许“自由定价”——这是一种极其浪漫的契约，你甚至可以免费下载到最高品质的 FLAC 或 WAV 无损文件。 对于小众音乐、电子和摇滚爱好者而言，这种体验是无与伦比的。你下载的不仅是一个几MB的音频文件，更是在进行一场审美的考古。当你顺着一个标签，在浩瀚的独立乐海里捞出一张惊艳内心的专辑时，那种多巴胺的分泌，与在旧书摊的角落里翻出一本绝版旧书的心情毫无二致。 Apple Music：主流流媒体的克制选择 当然，在这个时代完全与流媒体世界断联并不现实。在必须选择一个主流音乐平台时，我留给了 Apple Music 。 在所有充斥着社交动态、短视频弹窗和花哨皮肤的流媒体软件中，Apple Music 是极其克制的。它没有花哨的社交、没有各种广告，而且原生提供了近乎严苛的高解析度无损（Hi-Res Lossless）音质。更重要的一点，在于它对“母带和曲库”的工业级尊重。 它更像是我的“国家图书馆”。当我想听那些世界主流乐坛的经典之作、交响乐或者最新发行的正版专辑时，它是一个无比靠谱的云端支撑。在这里，我不需要担心粗劣的二次压缩，它以一种高级而安静的姿态，作为我本地资料库的一块巨大延伸。 foobar2000：数字手艺人的精神避难所 如果说云端的 Apple Music 和远方的 Bandcamp 解决了“水源”问题，那么 foobar2000 就是我精心建造的储水池。在这个池子里，所有的水，都必须打上我个人的烙印。 坚守本地播放，我图的就是它流淌在骨子里的三点优势： 极致的纯净： 当它在电脑上启动时，没有开屏雷击，没有“猜你喜欢”，只有一张干净的曲目列表。它把所有的舞台都留给了音乐本身。 恐怖的定制能力： 它是软件界的乐高。刚安装时它简陋得像 Windows 98 的表格，但通过组件的拼装，你可以像素级地定制歌词、专辑墙和音频输出。亲手配置它的过程，就像是在深夜书房里亲手整理自己的书架，给每一本书贴上标签、分类上架。 高保真音质： 通过 WASAPI 独占或 ASIO 输出，绕过操作系统的层层干预，将音符原汁原味地送入解碼器。这是对音乐创作者的尊重，也是对听觉的绝不妥协。 延伸：从“歌单”到“知识库”的同构 这种对本地软件的偏执、对模块化拼装的痴迷，其实早就不局限于我的音乐播放器了。当我回望自己的日常，我发现自己正将这种“手艺人”式的秩序感，全面推向我的数字世界。 听歌用 foobar2000，而阅读与写作，我则习惯交给 Emacs 里的 Org-mode 。 仔细想来，这两者的内核竟是如此的同构：它们在刚打开时都简陋得近乎硬核，需要你耐着性子去写配置、加插件、调参数；但它们都拥有“恐怖的定制能力”。你用代码和纯文本去搭建一个专属于自己的笔记系统，就像你用组件去魔改一个播放器界面一样。 ...

为什么要有自己的域名？

前言 截至今天，我终于把部署到 GitHub 上的 Hugo 静态博客挂到自己的域名下了。我的域名地址如下👇： fupengcheng.com 记录一些关于技术和乱七八糟的想法。 不再赘言，正题如下。 为什么需要有自己的域名？ 命名权即控制权 。 互联网的地址系统（DNS）本质上是一套“谁能被找到”的权力结构。比如，我在 github.io 、微信公众号、简书等平台产出内容时，我的“名字”（url）是平台赐予的，而不是自己的—— milteau.github.io 这个地址，本质上是归 GitHub 所有，我只能被允许“挂靠”在这个命名空间下。 脱离平台的“治外法权” 。数字主权的核心焦虑，是数字身份和数字资产，是否处于别人的司法管辖和商业条款下。域名让内容的所有权和平台的选择权，重新回到自己的手上。 例证：众多用户“逃离”老牌笔记软件 EverNote 的行为。（也包括我🤭） 抵抗中心化的“最小单元” 。 但是，域名不是终极主权。需要承认的是，域名系统（DNS）本身也是由 ICANN 和各国注册局管理的中心化基础设施——域名依然可能因为政策、法律纠纷、注册局的判断而被暂停解析。就像我在部署时，在解析设置的过程中，我配置好了一切，但是因为没有完成 未实名认证 导致全球查不到一样，这本身就是一种国家/平台对我们“数字地址”的管辖权体现。 最后 域名是个体在互联网上争取“可迁移的身份连续性”和“脱离单一平台锁定”的最小实践单元——它不能让你完全自治，但能让你在被某个平台抛弃时（比如之前 EverNote 的文件格式是 .enex ，但如今改成了 .note 格式。迁移繁琐，甚至迁移后，没有适配的软件打开。我能理解这样做是为了防止用户流失），但能让我们在被某个平台抛弃时，至少带着自己的名字重新开始。 附件

为什么我选择 Fastmail？

前言 前段时间我又去 Derek Sivers 的博客看他过去写的文章。有一篇叫做 Tech Independence 的文章，让我受益匪浅。 他的这篇文章实际上是技术型的内容，一步一步教你如何技术独立。实操性很强。 然而，我想要写得则是—— 为什么要技术独立？ 为什么要技术独立？ 技术独立意味着数字主权与数据隐私的回归。 在今天这个时代，我们的大部分数字生活都被寄托管在少数几个科技巨头的服务器上。免费的代价是昂贵的——我们交出了自己的行为数据、阅读偏好、甚至是一闪而过的想法。当一个服务是“免费”的时候，我们自己就成了商品。 而“技术独立”，就是要把这部分被出让的主权重新拿回来。这并不是说我们要偏执地去重复造轮子，或者彻底与现代网络隔离，而是要建立一种“有选择的掌控感”： 数字的所有权。 我的文字、我的笔记、我的邮件，应该以最通常的格式（比如纯文本的 Org/Markdown, 或者标准的 Mbox 格式）保存在我自己能控制的地方，而不是被锁死在某个特定软件的私有生活里。 迁移的自由度。 如果有一天某个服务商倒闭了，或者改变了隐私政策，我可以在半天之内带着我所有的数据，毫无痛感地迁移到下一个替代方案中。 这就是为什么我最终决定把“电子邮箱”这个互联网生活的核心锚点，从免费的公共大池子，搬到了 Fastmail。 顺便提一下，我很早就把笔记软件换成了自由软件 GNU Emacs。用 Emacs 的人大多有一种共识： 商业公司的寿命往往没有你的笔记长。 只要纯文本格式在，只要开源社区在，我的数字资产就是永存且绝对安全的。 而对待电子邮箱，我也顺着同样的逻辑去审视它。 为什么是电子邮箱？为什么是 Fastmail？ 电子邮箱是我们数字身份的底层资产。不管是银行账户、社交媒体还是各种生产力工具，几乎所有的线上服务都绑定在你的邮箱上。如果你的邮箱主权不在自己手里，所谓的“技术独立”就是空中楼阁。 我选择 Fastmail，恰恰是因为它在商业和技术的交界点上，满足了我对“掌控感”的苛求： 尊重开放协议。它是现代邮件协议 JMAP 的核心推动者，同时也完美支持 IMAP 和 CardDAV/CalDAV。这意味着什么？这意味着我可以用任何我喜欢的第三方开源客户端（甚至是在 Emacs 里用 mu4e 或 notmuch ）去无缝阅读和管理我的邮件。我的数据没有被任何高墙围起来。 真正的“付费购买尊严”。Gmail 很好用，但它是通过扫描你的邮件内容来精准投放广告的。在 Google 眼里，你不是客户，你是供广告商消费的“数据源”。 Fastmail 是一家纯粹的付费订阅制公司。我付钱，它提供顶级的邮件托管服务。这种极其古老且健康的商业模式，反而给了我最大的安全感—— 因为我知道，当我是消费者时，我的隐私就是它的产品护城河；它没有动机去窥探我的数字生活。 结语 从 GNU Emacs 到 Fastmail，表面上是在折腾工具，底层逻辑其实是一场 数字生活的精神断舍离。 技术独立不意味着你要活得像个苦行僧，每天去编译自己的操作系统。它意味着你在做选择时，多了一份“如果这个世界变坏了，我随时可以拔线走人”的底气。付费给 Fastmail，买下的不仅是一个稳定的邮件收发工具，更是买下了我对自己数字人生的解释权。

阅读随笔：我的读书经验浅谈

“在大审判的那天，当那些伟大的征服者、律师和政治家走上前去接受他们的奖赏——他们的皇冠、他们的桂冠、他们不朽的名字时，上帝看到我们胳膊下夹着书走来，会转过身去，不无欣羡地对彼得说：‘看，这些人不需要奖赏。我们这里没有什么东西可以给他们。他们一生热爱读书。’” —— 弗吉尼亚.伍尔夫《普通读者》 也不知多少年了，很少写散文的我，在今天这个仲夏之夜，又开始写一篇散文。我想写的是关于读书的。 许是因为“三十而立”，又或是因为“吾生也有涯，而知也无涯”，我重新回顾到阅读这条主线上来。其实说重新回归到阅读这条主线也不大妥帖，倒不如说，阅读其实从未离开过我。它更像是一条潜流，一艘夜航之船。在我被生活的琐碎、职业的身份、以及各种世俗系统裹挟着向前奔涌、穿行之时，始终在看不见的地底静静流淌、缓缓漫游。三十岁之前，我以为读书是在寻找岸。而在这个仲夏之夜，苍蝇嗡嗡闹耳，我猛然回头，才惊觉读书就是那条载我的河、搭我的船。—— 那些看似没有读书的日子，实则，我一直在用生活本身去咀嚼和消化过去读过的某一页书。 我在写这篇的时候，其实也是吃惊的。吃惊自己身份一直在变，从商业中的主管到销售，从销售到猎头，再从疫情时期的无业考编，到后疫情后的教育从业者。恰如鲍勃.齐蒙说的“人是流动的且嬗变的”。在那些高频转动的社会齿轮里，我曾以为自己快要被面目全非地拆解，像拆零件似的。在写字楼里谈论KPI、Pipeline，在未生计奔波的途中，我都曾以为自己弄丢了那个最初的自己。可直到今天动笔才发现，每当一个旧的身份碎裂、一个新的身份尚未建立的迷茫缝隙里，我全是在废墟中靠着几本书把灵魂重新拼凑起来的。职业和身份只是我在岸上穿上的不同戏装，而水面之下，支撑我没有溺亡的，依然是那些阅读留下的骸骨。 我注六经，六经注我。 —— 陆九渊 早年读书，大抵是“我注六经”的底色。总想着去皓首穷经，贪图阅读的数量，而非质量。而现在，我推崇一个“慢”字。慢意味着取舍与选择。在如今信息爆炸的时代，我选择了一种看似古典的方式——读经典。借用新东方联合创始人王强的话说：“读书只读第一流的书。”当然，这是一家之言。但不可否认，读经典会为我们节省时间，让慢读、再读有了更多的时间。因为，精神的重建不需要一万块劣质的砖头，只需要几根真正能承重的纯钢大梁。 最近，我就在慢读、再读。我读《诗经》，读一首，背一首。我读名著小说，比如 /Alice's Adventures in Wonderland/。有的是我之前读过的，但重新读一遍，背一遍，感受又是不一样的。为什么要背呢？用一个时髦 (modern) 的词——“让知识穿过身体”。 所以，我才意识到哪怕自己的外界角色如何流变，我骨子里仍是一个读书人。当然，我不仅有读书癖，也有猎书癖。 讲到猎书，我一般是按图索骥的。我很喜欢伍尔夫的意识流作品，比如今年读过的一本《达洛维夫人》，《伍尔夫读书笔记》。伍尔夫不仅仅是一个伟大的小说家，而且也是文学评论家，她的读书癖较之我而言，估计相差不大。虽然，我不清楚，但在这点上，我有我的自信。毛姆也一样。阅读是随身携带的避难所。这话说的不错。后来，我又陆续读了一些好书，比如陈彦的《主角》。这本书还在阅读，对了，还有米兰.昆德拉的《不能承受的生命之轻》。 《主角》里面的女主角原本叫易招弟，后来改名忆秦娥（因秦腔《忆秦娥.娄山关》而一战成名，走向舞台中央），再到最后改名易青娥。我从她三次更名中，看到一个生命在寻找岸、迷失于角色、最终靠筋骨立定的过程。这一过程，像极了我在那些职业名头更换的过程。 写到这，我发现，我没有像程序地写我如何如何读书，而是以一种感性的方式演化我的读书经验。在慢读、再读中…… 此刻，夜静谧。只想着星星地虫声。
