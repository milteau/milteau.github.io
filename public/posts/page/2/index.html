2011年秋天，广东省乌坎渔村发生了动乱。开发商与地方官员勾结夺走了农民的土地，然后取走了出售这些土地所获得的利益。最初是在9月份，几百名受到侵害的农民举行了相对小规模的抗议。到12月份，一名参加抗议的村民在被警方拘留期间死亡，势态随之升级为大规模的反抗。数天内，近两万名村民被鼓动参与。他们将官员赶出村子，设立了路障，用简单的武器武装自己。他们要求归还农田。 虽然中国的媒体被令停止播发任何有关的消息，许多中国人还是能通过互联网和国外媒体了解正在发生的情况。最终，广东省委副书记与抗议者会面，化解了此事。当地政府承认村民的要求是合法的，并将部分土地归还给村民，在抗议初期遭逮捕的村民也获得释放。随后村民举行了无记名投票的自由选举。抗议活动的一名主要组织者，获得了压倒性胜利，获选为新村长。那些希望看到中国民主改革的人，把乌坎发生的事看作是轰动的事件。 结论 一是共产党能保持它的掌控地位，在党的支持下恢复秩序。二是共产党可以运用软硬两手控制局势。在态势还没有升级前，它可以派非常强大的国家安全机器制止动乱，将问题控制在萌芽状态。另一方面它支持村民反对腐败的地方官。那种认为共产党彻底腐败的想法是过于简单了。事实上，在整个抗议过程中，乌坎村民一直小心地在横幅上表明，他们支持共产党，反对腐败的地方官。 政治变革 与一些欧元区国家相比，美国的债务和赤字问题看起来相对轻微。它的处境之所以比较好，部分原因是因为美元是世界储备货币，这意味着美国享有比其他国家低许多的借贷成本。但由于消费方向错误，情况已不容它自满。 除此之外还有一些问题： 全国性基础设施需要提升； 阶级鸿沟越来越大； 种族歧视的根深蒂固； 以及过分依赖金钱； 我所认知的美国 李光耀举例美国出兵阿富汗，除掉萨达姆等事件，把美国的外交政策方针剖析的深刻。两个字——自大。相反，中国则是更明智。 中国人不认为改变别人的制度关他们的事。制度怎么样，他们就怎么样去应对，并尽量从中获取好处，而不把自己套牢。美国人的问题是，他们带着相信自己有能力改变制度的想法介入，结果是一次又一次证明他们错了，他们没有改变世界。 欧洲：衰退与分歧 欧元的问题 欧元的根本问题是，如果财政尚未整合，就不可能实行货币一体化，尤其是当一个地区同时有德国与希腊这种消费和节俭习惯如此迥异的国家。这样的不协调最终将破坏整个制度。因此，欧元注定会困难重重，灭亡早已存于其基因。欧元过去几年的问题，不应被看作是因为一两个国家的政府做不到量入为出，或者是其他国家未能提醒它们入不敷出的危险。也就是说，欧元的困境并非只要是几个相关国家在实行过程中，作出了不同的且更负责任的决定，就可以避免的一起历史事件。相反的，这一历史的必然性只是在等待着某个时机发生。 这段对欧元问题分析，从历史角度切入，十分深刻。一言以蔽之： 李光耀看穿了欧洲一体化的核心障碍：欧洲各国无法放下各自几百上千年的民族国家历史、主权尊严与文化惯性，去真正融合成一个统一的“欧洲合众国”。 李光耀在这里极其精准地指出了欧元区在制度设计上的非理性。在传统的经济学逻辑中，货币政策（印钱、加息/降息）与财政政策（收税、花钱、借债）必须是捆绑在一起的。 然而，欧盟的设计却是一个异类： 它收走了各国的货币主权（统一用欧元，由欧洲央行控制），却把财政主权留给了各国政府。 这就导致了一个巨大的漏洞：福利国家可以为了选票大肆举债、挥霍无度，而一旦发生债务危机，由于没有本币贬值的手段来稀释债务，最终只能让整个欧元区为其买单。这种“风险共担，但权利分散”的体制，注定了它在面对危机时极度脆弱。 所例举德国、希腊两个不同国家迥异的消费习惯——隐喻欧洲内部两种完全无法兼容的经济文明形态。 重塑数字花园：我的现代 Hugo 与 GitHub Actions 实践

在这个信息过载、算法推荐横行的时代，拥有一个完全属于自己的独立数字花园，变得越来越珍贵。今天，这个完全由文本和纯粹代码构建的博客终于正式上线了。 不过，它的诞生并非一帆风顺，而是一场关于现代前端工具链和自动化部署的硬核排障之旅。 破局：从混乱到纯净 在搭建之初，我曾陷入过本地编译与多重本地 Git 嵌套的“死结”中。每当在本地运行 hugo 生成静态文件时，冗余的缓存与错综复杂的指针冲突就会接踵而至，甚至让部署陷入 404 的无底深渊。 在经历了数次删库、重来的阵痛后，我决定彻底重构博客的底层架构，确立了一条核心原则： 让工具回归纯粹，把复杂留给自动化。 1. 彻底斩断本地的“包袱” 我彻底清空了本地的 public 文件夹。在现代的静态网站工作流中，本地不应该包含任何编译后的“垃圾文件”，而应该只保留最纯粹的源码和文章本身。 2. 驯服 GitHub Actions 真正的解法在云端。通过配置 .github/workflows/hugo.yaml ，我将编译的重任完全交给了 GitHub 虚拟服务器。在这个过程中，虽然经历了一个错位的冒号（=:=）、官方旧版本 Hugo 的不兼容等小插曲，但当网页端最终跳出那个 治愈的绿色对勾（🟢 Success） 时，一切折腾都具有了意义。 优雅的工作流 现在，这个依托于 Hugo (PaperMod 主题) 与 GitHub Actions 的花园骨架前所未有的健康。我未来写文章的流程将变得无比纯粹与优雅： 打开我最钟爱的编辑器，用干净的 Org-mode 记录思考。 敲下三行最基础的命令： git add . git commit -m "feat: publish my new post" git push origin master 关掉终端，剩下的事情全部交给云端机器人。一分钟后，网站自动满血同步。 种下第一颗种子 这个博客不仅是一个发布文章的地方，更是我安放思考、记录阅读与翻译心得、沉淀知识的数字花园。它没有点赞的压力，没有流水的算法，只有文字最本真的流动。 感谢在排障过程中给予启发的伙伴，也感谢每一个卡在冒号和嵌套目录里的深夜。 ...

我的第一个org格式的博客

缘起 我跟Emacs的缘分有些年头了。最初了解Emacs还是奔着它“神之编辑器”的名头。后来，我尝试了一段时间后，又不用了。最近这段时间，我才开始用Emacs重新记录、整理我的读书笔记。越用越上手。Emacs中有一个包叫做denote，十分有用。我用它做卡片笔记。后来，随着我使用的投入深度，我又配置好了magit，再后来我用Emacs的Eshell，直接调用终端，用hugo搭建好了这个主题是PaperMod的静态页面博客。 为什么想到搭建这个博客 正如之前说的，我用Emacs中的Denote管理自己的笔记。之前读过一本书《聪明人如何做笔记》。实则聪明人做笔记的核心在于： 将知识转化为自己的思考，并建立知识网络。 正如这句话说的那样，我打算将自己碎片的笔记，由点到面，连成知识网络，于是有了这个博客。 后话 我其实很早之前就用hugo搭建过类似的静态博客，但总是卡在推送这一步。这次用Emacs就成功了。或许冥冥之中我跟这个工具有缘分。
